<h1 align="center">CALCULATING_OF_METER_READINGS</h1>

**CALCULATING_OF_METER_READINGS** — ежемесячный дорасчёт текущих показаний счётчиков TowerStore для автоматического расчёта текущих показаний по счётчикам на основе архивных, интегральных и периодических данных с учётом различных сценариев и приоритетных алгоритмов.

---

## 🚀 Как пользоваться проектом

1. Перейдите в папку `dist`.
2. Запустите файл main.exe.
3. Результаты расчёта будут сохранены в папке `data` в файле `calculation_results.xlsx`:
   - **calc** — итоговые рассчитанные показания;
   - **add** — дополнительная информация, использованная при расчётах.
4. При возникновении ошибок подробные отчёты будут сохранены в папке `logs`.

## ⚙️ Настройки проекта

Конфигурация выполняется через файл `.env`.

Доступные параметры:

| Переменная              | Описание                                              | Значение по умолчанию |
|-------------------------|-------------------------------------------------------|-----------------------|
| `AVG_EXP`               | Средний расход по опоре                               | `1200`                |
| `DEBUG`                 | Подробный вывод в консоль                             | `False`               |
| `ARCHIVE_EXTRA_COLUMNS` | Список столбцов из файла A для добавления в результат |                       |

## 📊 Входные данные

Для расчёта используются три файла из папки `data`:

| Обозначение | Файл                         |
|------------|-------------------------------|
| **A**      | `архив.xlsx`                  |
| **B**      | `интегральные_показания.xlsx` |
| **C**      | `показания_за_период.xlsx`    |

Файл **A** используется исключительно для добавления в данные файла **B** следующих полей по столбцу **Код ЭО**:
- АСКУЭ
- Шифр опоры

## 🧮 Алгоритмы дорасчёта текущих показаний

Расчёт выполняется **только при наличии и валидности** полей  
**действительно на1** и **действительно на2** в файле **C**.

Алгоритмы применяются **строго в указанном порядке приоритета**.

### 1️⃣ `base_algoritm`

Применяется, если:
- совпадают **«Номер ПУ»** и **«Шифр опоры»** в файлах **B** и **C**;
- заполнено поле **«Показания на конец периода»** в файле **C**.

#### Условие 1

Если:
- **действительно на1** — первое число предыдущего месяца;
- **действительно на2** — первое число текущего месяца,
- **Шифр опоры** — питается от шлейфа,
```
x = y
```
> где y — показания на конец периода из файла **C**

#### Условие 2

Если даты не соответствуют первым числам месяцев:
```
x = r * dt / n + y
```
> где: r — расход за период; n — количество дней между датами; dt — количество дней в месяце минус n; y — показания на конец периода.

---

### 2️⃣ `add_algoritm`

Применяется, если тип точки учёта:
- **Коммерческая** (приоритет),
- или **Контрольная**.

#### Условие 1

Если опора питается от шлейфа:
```
x = z
```
> z — предыдущие показания из файла **B**.

#### Условие 2

Если даты соответствуют первым числам месяцев:
```
x = r + z
```

#### Условие 3

Если даты не соответствуют первым числам месяцев:
```
x = r * dt / n + z
```

---

### 3️⃣ `extra_algoritm`

#### 3.1 Нет источника питания и опора нигде не используется как источник:
```
x = a * AVG_EXP + z
```
> где a — количество групп операторов (`Megafon`, `MTS`, `Tele2`, `Vimpelcom`); z — предыдущие показания.

#### 3.2 Опора — мастер-опора (источник питания для других опор):
```
x = b * AVG_EXP + z
```
> где b = сумма `a` по всем зависимым опорам + сама опора

#### 3.3 У опоры есть источник питания:
```
x = z
```

### Расчет расхода
Значение расхода вычисляется как текущие показания **x**, деленные на коэффициент
трансформации. Коэффициент берется из файла **B**. Если значение не указано, по умолчанию используется 1.


## 🛠️ Сборка проекта

Для создания исполняемого файла используется `pyinstaller`.

```bash
pyinstaller --onefile --icon=icon.ico \
  --add-data "data;data" \
  --add-data "logs;logs" \
  main.py
```
> В результате будет создан файл main.exe в папке dist.
